Introduction to HTML

HTML (HyperText Markup Language) is the standard language used to create web pages. It was created by Tim Berners-Lee in 1991 and has since evolved through various versions to become the backbone of the World Wide Web. HTML is used to structure content on the web, allowing developers to define elements such as headings, paragraphs, links, images, and more.

Timeline of HTML Development

  • 1991: HTML 1.0 - The first version of HTML was introduced by Tim Berners-Lee.
  • 1995: HTML 2.0 - This version included support for forms and tables.
  • 1997: HTML 3.2 - Introduced support for scripting languages like JavaScript and applets.
  • 1997: HTML 4.0 - Added support for CSS, improved accessibility, and introduced new elements.
  • 1999: HTML 4.01 - A minor update to HTML 4.0, fixing errors and improving standards compliance.
  • 2000: XHTML 1.0 - A reformulation of HTML 4.01 in XML, making it more strict and extensible.
  • 2008: HTML5 - The latest version, which includes new elements for multimedia, graphics, and more semantic tags.
  • 2014: HTML5.1 - An update to HTML5, adding new features and improvements.
  • 2016: HTML5.2 - Further updates to HTML5, including new elements and attributes.
  • Ongoing: HTML continues to evolve with new features and improvements being added regularly.

Basic Structure of an HTML Document

An HTML document typically consists of the following basic structure:

<!DOCTYPE html>
<html>
<head>
    <title>Page Title</title>
</head>
<body>
    <h1>This is a Heading</h1>
    <p>This is a paragraph.</p>
</body>
</html>

Key HTML Elements

  • <!DOCTYPE html>: Declares the document type and version of HTML.
  • <html>: The root element that contains all other HTML elements.
  • <head>: Contains meta-information about the document, such as the title and links to stylesheets.
  • <title>: Sets the title of the web page, which appears in the browser tab.
  • <body>: Contains the content of the web page, including text, images, links, etc.
  • <h1> to <h6>: Heading tags used to define headings of different levels.
  • <p>: Defines a paragraph of text.
  • <a>: Defines a hyperlink to another page or resource.
  • <img>: Embeds an image in the web page.
  • <div>: A container element used to group other elements for styling or layout purposes.

HTML5 Semantic Elements

HTML5 introduced several new semantic elements that provide better structure and meaning to web content. These elements help improve accessibility and SEO by clearly defining the purpose of different sections of a web page.

Structural Semantics

  • <header>: Represents the header section of a document or a section, typically containing introductory content or navigation links.
  • <footer>: Represents the footer section of a document or a section, typically containing information about the author, copyright, or related links.
  • <nav>: Defines a section of navigation links.
  • <main>: Represents the main content of the document, unique to the document and not repeated across pages.
  • <section>: Defines a thematic grouping of content, typically with a heading.
  • <article>: Represents a self-contained piece of content that could be distributed independently, such as a blog post or news article.
  • <aside>: Contains content that is tangentially related to the main content, such as sidebars or pull quotes.

Textual/Metadata Semantics

  • <h1> to <h6>: Heading tags used to define headings of different levels.
  • <p>: Defines a paragraph of text.
  • <mark>: Highlights or marks text for reference or emphasis.
  • <time>: Represents a specific point in time or a time duration, useful for marking up dates and times.
  • <address>: Provides contact information for the author or owner of a document or article.
  • <abbr>: Represents an abbreviation or acronym, providing a full description via the title attribute.
  • <cite>: Represents the title of a work, such as a book, article, or song.
  • <figure>: Used to encapsulate media content like images, diagrams, or code snippets, along with an optional caption defined by the <figcaption> element.
  • <figcaption>: Provides a caption or legend for the content within the <figure> element.

Form/Data Semantics

  • <label>: Defines a label for an input element, improving accessibility and usability.
  • <input>: Represents a variety of input controls (text, checkbox, radio, etc.) for user data entry.
  • <textarea>: Defines a multi-line text input control for user data entry.
  • <select>: Represents a drop-down list of options for user selection.
  • <option>: Defines an option within a & lt;select> element.
  • <datalist>: Provides a list of predefined options for an <input> element, allowing users to choose from a set of suggestions.
  • <output>: Represents the result of a calculation or user action, often used in forms.
  • <button>: Represents a clickable button.

Media Semantics

  • <audio>: Embeds an audio player in the web page, allowing for the inclusion of audio content.
  • <video>: Embeds a video player in the web page, allowing for the inclusion of video content.
  • <source>: Used within the <audio> and <video> elements to define multiple media sources for different formats or resolutions.
  • <track>: Provides text tracks (such as subtitles or captions) for <video> and <audio> elements.
  • <picture>: A container element that allows for multiple sources of an image, enabling responsive images that adapt to different screen sizes and resolutions.

Table/Data Semantics

  • <table>: Represents tabular data, consisting of rows and columns.
  • <caption>Represents table title or description.
  • <thead>: Defines the header section of a table, containing column headings.
  • <tbody>: Represents the main body of a table, containing data rows.
  • <tfoot>: Defines the footer section of a table, typically containing summary information.
  • <tr>: Represents a table row, containing one or more table cells.
  • <th>: Defines a header cell in a table, typically displayed in bold and centered.
  • <td>: Represents a standard data cell in a table.

Key Benefits of Semantic HTML5

  1. Accessibility: Screen readers and assistive technologies can better understand content.
  2. SEO: Search engines can better index and rank your content.
  3. Maintainability: Code is easier to read and manage.
  4. Consistency: Provides a standard way to structure content across websites.

HTML Attributes

  • They are always written in the start tag.
  • They usually come in name="value" pairs.
  • Attributes control the behavior, appearance, or metadata of elements.

Images

In HTML, images are added to web pages using the <img> tag. This tag is an empty (void) element, meaning it does not have a closing tag. The <img> tag requires at least two attributes: src and alt.

Basic Syntax:

<img src="image-url" alt="description">

Attributes:

  • src: Specifies the path to the image file (URL or local path).
  • alt: Provides alternative text for the image, used for accessibility and when the image cannot be displayed.
  • width: Sets the width of the image (in pixels or percentage).
  • height: Sets the height of the image (in pixels or percentage).
  • title: Adds a tooltip that appears when hovering over the image.

Example:

<img src="https://example.com/image.jpg" alt="Example Image" width="300" height="200" title="This is an example image">

Complete code for practice


<!DOCTYPE html>
<html>
<head>
    <title>HTML Images Example</title>
</head>
<body>
    <h2>HTML Images</h2>
    <img src="https://via.placeholder.com/300x200" alt="Placeholder Image" width="300" height="200" title="This is a placeholder image">
</body>
</html>

Tables

In HTML, tables are used to display data in a structured format using rows and columns. The main tags used to create tables are <table>, <tr>, <th>, and <td>.

Basic Syntax:

<table>
    <tr>
        <th>Header 1</th>   
        <th>Header 2</th>
    </tr>
    <tr>
        <td>Data 1</td>
        <td>Data 2</td>
    </tr>
</table>

Attributes:

  • border: Specifies the width of the table border.
  • border-collapse: Controls whether table borders are collapsed into a single border or separated.
  • cellpadding: Sets the space between cell content and cell borders.
  • cellspacing: Sets the space between individual table cells.
  • width: Defines the width of the table (in pixels or percentage).
  • height: Defines the height of the table (in pixels or percentage).
  • align: Aligns the table on the page (left, center, right).
  • bgcolor: Sets the background color of the table.
  • colspan: Merges multiple columns into a single cell.
  • rowspan: Merges multiple rows into a single cell.
  • thead, tbody, tfoot: Used to group table header, body, and footer sections for better organization and styling.
  • caption: Adds a title or caption to the table.
  • style: Allows for inline CSS styling of the table.
  • summary: Provides a summary of the table's purpose and structure for accessibility.
  • title: Adds a tooltip that appears when hovering over the table.

Example:

<table border="1" cellpadding="5" cellspacing="0" width="50%">
    <tr>
        <th>Name</th>
        <th>Age</th>
    </tr>
    <tr>
        <td>Alice</td>
        <td>30</td> 
    </tr>
    <tr>
        <td>Bob</td>
        <td>25</td> 
    </tr>
</table>

Iframes

In HTML, an iframe (inline frame) is used to embed another HTML document within the current document. The <iframe> tag creates a rectangular area on the page where the embedded content is displayed.

Basic Syntax:

<iframe src="url" width="width" height="height"></iframe>

Attributes:

  • src: Specifies the URL of the page to embed.
  • width: Sets the width of the iframe (in pixels or percentage).
  • height: Sets the height of the iframe (in pixels or percentage).
  • frameborder: Defines whether to display a border around the iframe (0 for no border, 1 for border).
  • allowfullscreen: Allows the iframe content to be displayed in fullscreen mode.
  • loading: Specifies whether to load the iframe content lazily or eagerly (values: "lazy", "eager").

Example:

<iframe src="https://www.example.com" width="600" height="400" frameborder="0" allowfullscreen></iframe>

Complete code for practice


<!DOCTYPE html>
<html>
<head>
    <title>HTML Iframes Example</title>
</head>
<body>
    <h2>HTML Iframes</h2>
    <iframe src="https://www.example.com" width="600" height="400" frameborder="0" allowfullscreen></iframe>
</body>
</html>

Media

In HTML, media elements are used to embed audio and video content into web pages. The main tags used for media are <audio> and <video>.

Audio Element:

<audio controls>
    <source src="audio-file.mp3" type="audio/mpeg">

<video width="640" height="360" controls>

</video>

Attributes:

  • width and height: Sets the dimensions of the video player.

<audio controls>

</audio>

Colors

In HTML, colors can be specified using various methods, including color names, hexadecimal values, RGB values, HSL values, and CSS variables. Colors are commonly applied to text, backgrounds, borders, and other elements using the style attribute or CSS.

Common Methods to Specify Colors:

Method Description Example
Color Names Using predefined color names (e.g., red, blue, green). style="color: red;"
Hexadecimal Values Using hex codes to define colors (e.g., #FF0000 for red). style="color: #FF0000;"
RGB Values Using RGB format to define colors (e.g., rgb(255, 0, 0) for red). style="color: rgb(255, 0, 0);"
HSL Values Using HSL format to define colors (e.g., hsl(0, 100%, 50%) for red). style="color: hsl(0, 100%, 50%);"

Gradients

Gradients are smooth transitions between two or more colors. In HTML/CSS, gradients can be created using the background-image property with the linear-gradient or radial-gradient functions.

Gradient Color Codes

Gradient Description
Linear Gradient from Red to Yellow
Linear Gradient from Blue to Green
Radial Gradient from Purple to Pink
Radial Gradient from Orange to Yellow

Example:

/* Linear Gradient */
background-image: linear-gradient(to right, red, yellow);   
/* Radial Gradient */
background-image: radial-gradient(circle, blue, green);

Applying Colors:

Colors can be applied to various HTML elements using the style attribute. Here are some examples:

Example:

<h1 style="color: blue;">This is a blue heading</h1>
<p style="background-color: #FFFF00;">This paragraph has a yellow background.</p>
<div style="border: 2px solid rgb(0, 255, 0);">This div has a green border.</div>
<span style="color: hsl(240, 100%, 50%);">This text is blue using HSL.</span>
